# 📌 Problem Statement

### Students Table

| Column Name | Type |
|------------|------|
| student_id | int |
| student_name | varchar |

- `student_id` is the primary key.
- Contains student information.

---

### Subjects Table

| Column Name | Type |
|------------|------|
| subject_name | varchar |

- `subject_name` is the primary key.
- Contains all subjects offered by the school.

---

### Examinations Table

| Column Name | Type |
|------------|------|
| student_id | int |
| subject_name | varchar |

- Contains exam attendance records.
- May contain duplicate rows.
- Each row indicates that a student attended a particular subject exam.

---

# 🎯 Task

For **every student** and **every subject**:

Find how many times the student attended that subject's exam.

Return:

- `student_id`
- `student_name`
- `subject_name`
- `attended_exams`

Ordered by:

```sql
student_id,
subject_name
```

---

# 💻 My SQL Solution

```sql
SELECT
    s.student_id,
    s.student_name,
    sub.subject_name,
    COUNT(e.subject_name) AS attended_exams
FROM Students s
CROSS JOIN Subjects sub
LEFT JOIN Examinations e
    ON s.student_id = e.student_id
    AND sub.subject_name = e.subject_name
GROUP BY
    s.student_id,
    s.student_name,
    sub.subject_name
ORDER BY
    s.student_id,
    sub.subject_name;
```

---

# 🧠 Explanation of My Approach

The problem requires:

✅ Every student

AND

✅ Every subject

Even if the student never attended that exam.

Therefore:

1. Generate all possible Student–Subject combinations.
2. Match attendance records.
3. Count how many times the student attended each subject.

---

# Step 1️⃣ Generate All Student-Subject Pairs

```sql
CROSS JOIN Subjects
```

This creates every possible combination.

---

### Students

| student_id | student_name |
|------------|--------------|
| 1 | Alice |
| 2 | Bob |

### Subjects

| subject_name |
|--------------|
| Math |
| Physics |

---

### After CROSS JOIN

| student_id | student_name | subject_name |
|------------|--------------|--------------|
| 1 | Alice | Math |
| 1 | Alice | Physics |
| 2 | Bob | Math |
| 2 | Bob | Physics |

Now every student appears with every subject.

---

# Step 2️⃣ Match Exam Records

```sql
LEFT JOIN Examinations e
```

Match:

```sql
s.student_id = e.student_id
AND
sub.subject_name = e.subject_name
```

This finds attendance records.

---

# Step 3️⃣ Count Attendance

```sql
COUNT(e.subject_name)
```

Counts how many matching exam records exist.

If no records exist:

```text
COUNT = 0
```

---

# Step 4️⃣ Group Results

```sql
GROUP BY
    s.student_id,
    s.student_name,
    sub.subject_name
```

Creates one row per student-subject pair.

# ⚠ Important SQL Concept

### Why CROSS JOIN?

The question asks for:

> Every student × Every subject

Even if no exam was attended.

Only `CROSS JOIN` generates all possible combinations.

---

### Why LEFT JOIN?

Without `LEFT JOIN`, students with zero attendance would disappear.

Using:

```sql
LEFT JOIN
```

keeps those rows and allows:

```text
COUNT = 0
```

---

# ⏱ Complexity Analysis

| Operation | Complexity |
|------------|-------------|
| CROSS JOIN | O(S × Sub) |
| LEFT JOIN | O(E) |
| GROUP BY | O(S × Sub) |

Where:

- `S` = Number of Students
- `Sub` = Number of Subjects
- `E` = Number of Examination Records
