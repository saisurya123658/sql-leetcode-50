# 📌 Problem Statement

### Employee Table

| Column Name | Type |
|------------|------|
| empId | int |
| name | varchar |
| supervisor | int |
| salary | int |

### 📝 Description

- `empId` is unique for each employee.
- Contains employee details such as:
  - Employee ID
  - Name
  - Supervisor ID
  - Salary

---

### Bonus Table

| Column Name | Type |
|------------|------|
| empId | int |
| bonus | int |

### 📝 Description

- `empId` is unique.
- Stores the bonus amount received by employees.

---

# 🎯 Task

Find employees who satisfy either:

✅ Bonus is less than `1000`

OR

✅ Employee did not receive any bonus

Return:

- `name`
- `bonus`

in any order.

---

# 💻 My SQL Solution

```sql
SELECT e.name, b.bonus
FROM Employee e
LEFT JOIN Bonus b
ON e.empId = b.empId
WHERE b.bonus < 1000
   OR b.bonus IS NULL;
```

---

# 🧠 Explanation of My Approach

The bonus information is stored in a separate table.

To include employees who did not receive a bonus, we use a **LEFT JOIN**.

---

## Step 1️⃣ Start with Employee Table

```sql
FROM Employee e
```

This table contains all employees.

---

## Step 2️⃣ Join Bonus Table

```sql
LEFT JOIN Bonus b
ON e.empId = b.empId
```

This matches employees with their bonus records.

If no matching bonus exists:

```text
bonus = NULL
```

---

## Step 3️⃣ Apply Conditions

```sql
WHERE b.bonus < 1000
OR b.bonus IS NULL
```

Include employees:

- Having bonus less than 1000
- Having no bonus record

# ⚠ Important SQL Concept

### Why Use LEFT JOIN?

If we use:

```sql
INNER JOIN
```

employees without bonus records would be excluded.

But the problem specifically asks to include employees who did not receive a bonus.

Therefore:

```sql
LEFT JOIN
```

is required.
