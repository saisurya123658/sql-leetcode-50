# 📌 Problem Statement

Table: `Views`

| Column Name | Type |
|-------------|------|
| article_id | int |
| author_id | int |
| viewer_id | int |
| view_date | date |

### 📝 Description

- The table may contain duplicate rows.
- Each row indicates:
  - Which article was viewed
  - Who authored the article
  - Who viewed the article
  - Date of view
- If `author_id = viewer_id`, it means the author viewed their own article.

---

# 🎯 Task

Find all authors who viewed at least one of their own articles.

Return the result table:

✅ With column name `id`  
✅ Sorted in ascending order

---

# 💻 My SQL Solution

```sql
SELECT DISTINCT author_id AS id
FROM Views
WHERE author_id = viewer_id
ORDER BY id;
```

---

# 🧠 Explanation of My Approach

To solve this problem, I filtered rows where the author viewed their own article.

---

## Step 1️⃣ Select Author IDs

```sql
SELECT DISTINCT author_id AS id
```

- `author_id` is renamed to `id`
- `DISTINCT` removes duplicate author IDs

---

## Step 2️⃣ Read Data From Table

```sql
FROM Views
```

The data is fetched from the `Views` table.

---

## Step 3️⃣ Find Self-Views

```sql
WHERE author_id = viewer_id
```

This condition checks whether:

- The viewer is the same person as the author

If true, that author viewed their own article.

---

## Step 4️⃣ Sort the Result

```sql
ORDER BY id;
```

Sorts the IDs in ascending order.

# ⚠ Important SQL Concept

## Why Use `DISTINCT`?

The table may contain duplicate rows.

Example:

```text
author_id = 4
viewer_id = 4
```

appears multiple times.

Without `DISTINCT`, duplicate IDs would appear in the result.
