# 📌 Problem Statement

Table: `Customer`

| Column Name | Type |
|-------------|------|
| id | int |
| name | varchar |
| referee_id | int |

### 📝 Description

- `id` is the primary key.
- Each row contains:
  - Customer ID
  - Customer name
  - Referee ID (who referred the customer)

---

# 🎯 Task

Find the names of customers who are:

✅ Referred by someone with `id != 2`  
OR  
✅ Not referred by anyone (`NULL`)

Return the result table in **any order**.

---

# 💻 My SQL Solution

```sql
SELECT name
FROM Customer
WHERE referee_id <> 2
OR referee_id IS NULL;
```

---

# 🧠 Explanation of My Approach

To solve this problem, I filtered the rows using the `WHERE` clause.

### Step-by-Step:

### 1️⃣ Select Required Column

```sql
SELECT name
```

This returns only customer names.

---

### 2️⃣ Read Data From Table

```sql
FROM Customer
```

The data is fetched from the `Customer` table.

---

### 3️⃣ Apply Conditions

```sql
WHERE referee_id <> 2
OR referee_id IS NULL;
```

This checks two conditions:

- Customers whose referee ID is **not equal to 2**
- Customers whose referee ID is `NULL`

The `OR` operator ensures rows satisfying at least one condition are returned.

# ⚠ Important SQL Concept

In SQL:

```sql
NULL <> 2
```

does **NOT** return `TRUE`.

So we must explicitly check:

```sql
referee_id IS NULL
```

to include customers without referees.
