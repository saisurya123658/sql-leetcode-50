# 📌 Problem Statement

### Users Table

| Column Name | Type |
|------------|------|
| user_id | int |
| user_name | varchar |

- `user_id` is the primary key.
- Stores all users.

---

### Register Table

| Column Name | Type |
|------------|------|
| contest_id | int |
| user_id | int |

- `(contest_id, user_id)` is the primary key.
- Stores contest registrations.

---

# 🎯 Task

For each contest, calculate:

```text
Percentage of Users Registered
=
(Number of Users Registered in Contest / Total Users)
× 100
```

Requirements:

✅ Round to 2 decimal places

✅ Sort by percentage (descending)

✅ If percentages are equal, sort by contest_id (ascending)

---

# 💻 My SQL Solution

```sql
SELECT
    contest_id,
    ROUND(
        COUNT(DISTINCT user_id) * 100 /
        (
            SELECT COUNT(user_id)
            FROM Users
        ),
        2
    ) AS percentage
FROM Register
GROUP BY contest_id
ORDER BY
    percentage DESC,
    contest_id;
```

---

# 🧠 Explanation of My Approach

To calculate contest participation percentage:

1. Count users registered in each contest.
2. Count total users.
3. Apply the percentage formula.
4. Round the answer.
5. Sort the result.

\[

# ⚠ Important SQL Concept

### Why Use DISTINCT?

```sql
COUNT(DISTINCT user_id)
```

Ensures each user is counted only once per contest.

Even if duplicate registrations existed, the count remains correct.

---

### Why Use a Subquery?

```sql
(
 SELECT COUNT(user_id)
 FROM Users
)
```

Provides the total number of users, which acts as the denominator for every contest.
