# 📌 Problem Statement

### Employees Table

| Column Name | Type |
|------------|------|
| id | int |
| name | varchar |

- `id` is the primary key.
- Each row contains an employee's ID and name.

### EmployeeUNI Table

| Column Name | Type |
|------------|------|
| id | int |
| unique_id | int |

- `(id, unique_id)` is the primary key.
- Stores the unique identifier corresponding to an employee.

---

# 🎯 Task

Display:

- `unique_id`
- `name`

For every employee.

If an employee does not have a unique ID, display **NULL**.

Return the result table in any order.

---

# 💻 My SQL Solution

```sql
SELECT unique_id, name
FROM Employees
LEFT JOIN EmployeeUNI
ON Employees.id = EmployeeUNI.id;
```

---

# 🧠 Explanation of My Approach

The problem asks us to show all employees, even those who do not have a matching record in the `EmployeeUNI` table.

Therefore, we use a **LEFT JOIN**.

---

## Step 1️⃣ Start with Employees Table

```sql
FROM Employees
```

The Employees table contains all employees.

---

## Step 2️⃣ Join EmployeeUNI Table

```sql
LEFT JOIN EmployeeUNI
ON Employees.id = EmployeeUNI.id
```

This matches employee IDs from both tables.

---

## Step 3️⃣ Why LEFT JOIN?

A `LEFT JOIN` returns:

✅ All rows from the left table (`Employees`)

✅ Matching rows from the right table (`EmployeeUNI`)

✅ NULL when no match exists

# 🔥 Why INNER JOIN Fails?

If we use:

```sql
SELECT unique_id, name
FROM Employees
INNER JOIN EmployeeUNI
ON Employees.id = EmployeeUNI.id;
```

Output would be:

| unique_id | name |
|------------|----------|
| 2 | Meir |
| 3 | Winston |
| 1 | Jonathan |

❌ Alice and Bob would be missing.

Because INNER JOIN only returns matching rows.

# ⏱ Complexity Analysis

| Operation | Complexity |
|------------|-------------|
| Join Operation | O(n + m) |

Where:

- `n` = rows in Employees
- `m` = rows in EmployeeUNI

---

# 🎓 Key Learning

Use **LEFT JOIN** when:

- You want all records from the left table.
- Matching records from the right table are optional.
- Missing matches should appear as `NULL`.
