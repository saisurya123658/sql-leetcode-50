# 📌 Problem Statement

### Weather Table

| Column Name | Type |
|------------|------|
| id | int |
| recordDate | date |
| temperature | int |

### 📝 Description

- `id` is unique for each record.
- No two rows have the same `recordDate`.
- Each row stores the temperature recorded on a specific date.

---

# 🎯 Task

Find all IDs where:

✅ The temperature is higher than the previous day (yesterday).

Return the result table in any order.

---

# 💻 My SQL Solution

```sql
SELECT
    w1.id
FROM Weather w1
JOIN Weather w2
ON DATEDIFF(w1.recordDate, w2.recordDate) = 1
WHERE w1.temperature > w2.temperature;
```

---

# 🧠 Explanation of My Approach

To compare today's temperature with yesterday's temperature, we need to compare rows within the same table.

This is achieved using a **Self Join**.

---

## Step 1️⃣ Create Two Copies of Weather Table

```sql
Weather w1
Weather w2
```

- `w1` → Current day
- `w2` → Previous day

---

## Step 2️⃣ Match Consecutive Dates

```sql
ON DATEDIFF(w1.recordDate, w2.recordDate) = 1
```

This ensures:

```text
w1.recordDate = w2.recordDate + 1 day
```

Example:

| Current Day | Previous Day |
|------------|-------------|
| 2015-01-02 | 2015-01-01 |
| 2015-01-03 | 2015-01-02 |
| 2015-01-04 | 2015-01-03 |

---

## Step 3️⃣ Compare Temperatures

```sql
WHERE w1.temperature > w2.temperature
```

Keep only rows where today's temperature is greater than yesterday's temperature.

---

# 📊 Example

## 📥 Input

### Weather Table

| id | recordDate | temperature |
|----|------------|------------|
| 1 | 2015-01-01 | 10 |
| 2 | 2015-01-02 | 25 |

| 4 | 2015-01-04 | 30 |

---

# 📖 Explanation

### Day 1 → Day 2

| Date | Temperature |
|--------|------------|
| 2015-01-01 | 10 |
| 2015-01-02 | 25 |

```text
25 > 10
```

✅ Include ID = 2

---

### Day 2 → Day 3

| Date | Temperature |
|--------|------------|
| 2015-01-02 | 25 |
| 2015-01-03 | 20 |

```text
20 < 25
```

❌ Exclude ID = 3

---

### Day 3 → Day 4

| Date | Temperature |
|--------|------------|
| 2015-01-03 | 20 |
| 2015-01-04 | 30 |

```text
30 > 20
```

✅ Include ID = 4

# ⚠ Important SQL Concept

## What is a Self Join?

A Self Join is when a table joins with itself.

```sql
FROM Weather w1
JOIN Weather w2
```

Used when comparing rows within the same table.

Common use cases:

- Compare current row with previous row
- Compare employees with managers
- Compare dates and events

---

# ⏱ Complexity Analysis

| Operation | Complexity |
|------------|-------------|
| Self Join | O(n²) |

Where `n` is the number of rows in the Weather table.
