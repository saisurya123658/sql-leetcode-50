# 📌 Problem Statement

Table: `World`

| Column Name | Type |
|-------------|------|
| name | varchar |
| continent | varchar |
| area | int |
| population | int |
| gdp | bigint |

### 📝 Description

- `name` is the primary key.
- Each row contains information about:
  - Country name
  - Continent
  - Area
  - Population
  - GDP

---

# 🎯 Task

A country is considered **big** if:

✅ Its area is at least `3000000` km²  
OR  
✅ Its population is at least `25000000`

Return:

- `name`
- `population`
- `area`

of all big countries.

Return the result table in **any order**.

---

# 💻 My SQL Solution

```sql
SELECT name, population, area
FROM World
WHERE area >= 3000000
OR population >= 25000000;
```

---

# 🧠 Explanation of My Approach

To solve this problem, I filtered countries based on the given conditions.

---

## Step 1️⃣ Select Required Columns

```sql
SELECT name, population, area
```

This returns only:

- Country name
- Population
- Area

---

## Step 2️⃣ Read Data From Table

```sql
FROM World
```

The data is fetched from the `World` table.

---

## Step 3️⃣ Apply Conditions

```sql
WHERE area >= 3000000
OR population >= 25000000;
```

This checks:

- Countries with area greater than or equal to `3000000`
- OR countries with population greater than or equal to `25000000`

The `OR` operator ensures countries satisfying at least one condition are included.
