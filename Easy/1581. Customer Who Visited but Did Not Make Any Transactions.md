# 📌 Problem Statement

### Visits Table

| Column Name | Type |
|------------|------|
| visit_id | int |
| customer_id | int |

- `visit_id` is unique.
- Stores information about customers visiting the mall.

---

### Transactions Table

| Column Name | Type |
|------------|------|
| transaction_id | int |
| visit_id | int |
| amount | int |

- `transaction_id` is unique.
- Stores transactions made during a visit.

---

# 🎯 Task

Find:

- `customer_id`
- Number of visits without any transactions (`count_no_trans`)

Return the result table in any order.

---

# 💻 My SQL Solution

```sql
SELECT
    customer_id,
    COUNT(visit_id) AS count_no_trans
FROM Visits
WHERE visit_id NOT IN
(
    SELECT visit_id
    FROM Transactions
)
GROUP BY customer_id;
```

---

# 🧠 Explanation of My Approach

The idea is simple:

1. Find visits that do not appear in the `Transactions` table.
2. Group those visits by customer.
3. Count how many such visits each customer made.

---

## Step 1️⃣ Find Visits Without Transactions

```sql
SELECT visit_id
FROM Transactions
```

Returns all visits where at least one transaction occurred.

Example:

| visit_id |
|----------|
| 1 |
| 2 |
| 5 |

---

## Step 2️⃣ Exclude Those Visits

```sql
WHERE visit_id NOT IN
(
    SELECT visit_id
    FROM Transactions
)
```

Removes all visits that had transactions.

Remaining visits:

| visit_id |
|----------|
| 4 |
| 6 |
| 7 |
| 8 |

---

## Step 3️⃣ Group by Customer

```sql
GROUP BY customer_id
```

Groups all remaining visits by customer.

---

## Step 4️⃣ Count Visits

```sql
COUNT(visit_id)
```

Counts the number of transaction-free visits per customer.

# ⚠ Important SQL Concept

## Why `NOT IN`?

```sql
WHERE visit_id NOT IN (...)
```

Returns visits that never appear in the Transactions table.

This helps identify customers who visited but did not purchase anything.
