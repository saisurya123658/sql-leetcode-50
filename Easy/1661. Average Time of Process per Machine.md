# 📌 Problem Statement

### Activity Table

| Column Name | Type |
|------------|------|
| machine_id | int |
| process_id | int |
| activity_type | enum |
| timestamp | float |

### 📝 Description

- `(machine_id, process_id, activity_type)` is the primary key.
- `activity_type` can be:
  - `'start'`
  - `'end'`
- Every process has exactly:
  - One start timestamp
  - One end timestamp
- Start time is always earlier than end time.

---

# 🎯 Task

For each machine:

1. Calculate the time taken for every process:

```text
Processing Time = End Timestamp - Start Timestamp
```

2. Compute the average processing time of all processes run on that machine.

3. Round the result to **3 decimal places**.

Return:

- `machine_id`
- `processing_time`

---

# 💻 My SQL Solution

```sql
SELECT
    a.machine_id,
    ROUND(AVG(b.timestamp - a.timestamp), 3) AS processing_time
FROM Activity a,
     Activity b
WHERE
    a.machine_id = b.machine_id
AND a.process_id = b.process_id
AND a.activity_type = 'start'
AND b.activity_type = 'end'
GROUP BY machine_id;
```

---

# 🧠 Explanation of My Approach

The start and end timestamps are stored in different rows.

To calculate the processing time:

- Match each `start` row with its corresponding `end` row.
- Calculate:

```text
end_time - start_time
```

- Find the average for each machine.

---

## Step 1️⃣ Self Join the Table

```sql
Activity a,
Activity b
```

We use the same table twice:

| Alias | Purpose |
|---------|---------|
| a | Start Activity |
| b | End Activity |

---

## Step 2️⃣ Match Same Machine and Process

```sql
a.machine_id = b.machine_id
AND
a.process_id = b.process_id
```

This ensures:

```text
Start Row ↔ End Row
```

belong to the same process.

---

## Step 3️⃣ Match Start and End Rows

```sql
a.activity_type = 'start'
AND
b.activity_type = 'end'
```

Now we have:

| Start Time | End Time |
|------------|----------|
| 0.712 | 1.520 |
| 3.140 | 4.120 |

for each process.

---

## Step 4️⃣ Calculate Process Time

```sql
b.timestamp - a.timestamp
```

Formula:

```text
Processing Time = End - Start
```

Example:

```text
1.520 - 0.712 = 0.808
```

---

## Step 5️⃣ Find Average Time

```sql
AVG(b.timestamp - a.timestamp)
```

Calculates average process duration per machine.

---

## Step 6️⃣ Round Result

```sql
ROUND(..., 3)
```

Rounds answer to 3 decimal places.

# ⚠ Important SQL Concept

### Why Self Join?

The start and end timestamps are stored in different rows.

To calculate duration:

```text
End Timestamp - Start Timestamp
```

we must join the table with itself.

---

# ⏱ Complexity Analysis

| Operation | Complexity |
|------------|-------------|
| Self Join | O(n²) |
| Grouping | O(n) |

Overall:

```text
O(n²)
```

Where `n` is the number of rows in the Activity table.
