# 📌 Problem Statement

### Sales Table

| Column Name | Type |
|------------|------|
| sale_id | int |
| product_id | int |
| year | int |
| quantity | int |
| price | int |

- `(sale_id, year)` is the primary key.
- `product_id` is a foreign key referencing the `Product` table.
- Each row represents a sale of a product in a specific year.

---

### Product Table

| Column Name | Type |
|------------|------|
| product_id | int |
| product_name | varchar |

- `product_id` is the primary key.
- Contains the name of each product.

---

# 🎯 Task

Report:

- `product_name`
- `year`
- `price`

for each sale recorded in the `Sales` table.

Return the result table in **any order**.

---

# 💻 My SQL Solution

```sql
SELECT product_name, year, price
FROM Sales
JOIN Product
ON Sales.product_id = Product.product_id;
```

---

# 🧠 Explanation of My Approach

The required columns are stored in two different tables:

- `product_name` → Product table
- `year`, `price` → Sales table

Therefore, we need to combine both tables using a **JOIN**.

---

## Step 1️⃣ Start With Sales Table

```sql
FROM Sales
```

The Sales table contains:

- Product ID
- Year
- Price

---

## Step 2️⃣ Join Product Table

```sql
JOIN Product
ON Sales.product_id = Product.product_id
```

This matches each sale with its corresponding product.

---

## Step 3️⃣ Select Required Columns

```sql
SELECT product_name, year, price
```

Returns:

- Product Name
- Year of Sale
- Price

---

# ⚠ Why JOIN Is Needed?

Without joining the tables:

- Sales table does not contain `product_name`
- Product table does not contain `year` or `price`

We need a JOIN to combine information from both tables.

---

# ⏱ Complexity Analysis

| Operation | Complexity |
|------------|-------------|
| JOIN Operation | O(n + m) |

Where:

- `n` = rows in Sales table
- `m` = rows in Product table

---

# 🎓 Key Learning

Use an **INNER JOIN** when:

- Related data is stored in multiple tables.
- Matching records exist in both tables.
- You need columns from both tables in the result.
