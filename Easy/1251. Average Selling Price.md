# 📌 Problem Statement

### Prices Table

| Column Name | Type |
|------------|------|
| product_id | int |
| start_date | date |
| end_date | date |
| price | int |

### 📝 Description

- Stores the price of a product during a specific date range.
- Price periods for the same product never overlap.

---

### UnitsSold Table

| Column Name | Type |
|------------|------|
| product_id | int |
| purchase_date | date |
| units | int |

### 📝 Description

- Stores product sales.
- Contains:
  - Product ID
  - Purchase Date
  - Units Sold

---

# 🎯 Task

For each product calculate:

```text
Average Selling Price
=
Total Revenue / Total Units Sold
```

Where:

```text
Revenue = Price × Units Sold
```

Requirements:

✅ Round to 2 decimal places

✅ If a product has no sales, average_price = 0

---

# 💻 My SQL Solution

```sql
SELECT
    p.product_id,
    ROUND(
        IFNULL(
            SUM(u.units * p.price) / SUM(u.units),
            0
        ),
        2
    ) AS average_price
FROM Prices p
LEFT JOIN UnitsSold u
ON p.product_id = u.product_id
AND u.purchase_date BETWEEN p.start_date AND p.end_date
GROUP BY p.product_id;
```

---

# 🧠 Explanation of My Approach

To calculate the average selling price:

1. Match each sale with the correct price period.
2. Calculate revenue for each sale.
3. Sum all revenues.
4. Divide by total units sold.
5. Round the answer.

---

# Step 1️⃣ Match Sales with Correct Price

```sql
LEFT JOIN UnitsSold u
ON p.product_id = u.product_id
AND u.purchase_date
BETWEEN p.start_date AND p.end_date
```

This ensures each sale gets the correct price valid on that purchase date.

# ⚠ Important SQL Concept

### Why LEFT JOIN?

The problem states:

> If a product has no sales, average_price = 0

Using:

```sql
LEFT JOIN
```

ensures products with no sales still appear in the result.

---

### Why IFNULL()?

```sql
IFNULL(expression, 0)
```

Without sales:

```sql
SUM(...) = NULL
```

We convert it to:

```text
0
```
