# 📌 Problem Statement

### Queries Table

| Column Name | Type |
|------------|------|
| query_name | varchar |
| result | varchar |
| position | int |
| rating | int |

### 📝 Description

- The table may contain duplicate rows.
- `position` ranges from **1 to 500**.
- `rating` ranges from **1 to 5**.
- A query is considered **poor** if:

```text
rating < 3
```

---

# 🎯 Task

For each `query_name`, calculate:

### 1️⃣ Query Quality

```text
Quality
=
Average of (rating / position)
```

### 2️⃣ Poor Query Percentage

```text
Poor Query Percentage
=
(Poor Queries / Total Queries)
× 100
```

Where:

```text
Poor Query = rating < 3
```

Requirements:

✅ Round both values to 2 decimal places.

---

# 💻 My SQL Solution

```sql
SELECT
    query_name,
    ROUND(AVG(rating / position), 2) AS quality,
    ROUND(
        AVG(
            CASE
                WHEN rating < 3 THEN 1
                ELSE 0
            END
        ) * 100,
        2
    ) AS poor_query_percentage
FROM Queries
GROUP BY query_name;
```

---

# 🧠 Explanation of My Approach

We need two metrics for each query:

1. Quality
2. Poor Query Percentage

Both can be calculated using aggregate functions.

---

# Step 1️⃣ Calculate Query Quality

Formula:

\[
\text{Quality}
=
\text{AVG}\left(\frac{\text{rating}}{\text{position}}\right)
\]

:contentReference[oaicite:0]{index=0}

SQL:

```sql
AVG(rating / position)
```

---

# Step 2️⃣ Identify Poor Queries

A poor query is:

```sql
rating < 3
```

Convert to:

| Rating | Value |
|---------|--------|
| < 3 | 1 |
| >= 3 | 0 |

Using:

```sql
CASE
WHEN rating < 3 THEN 1
ELSE 0
END
```

---

# Step 3️⃣ Calculate Poor Query Percentage

Formula:

\[
\frac{\text{Poor Queries}}
{\text{Total Queries}}
\times100
\]

Instead of:

```sql
SUM(...) / COUNT(*)
```

we use:

```sql
AVG(...)
```

because:

```text
AVG(1,0,0)
=
1/3
```

which directly gives the percentage ratio.

SQL:

```sql
AVG(
 CASE WHEN rating < 3 THEN 1
 ELSE 0
 END
) * 100
```

---

# Step 4️⃣ Round Results

```sql
ROUND(..., 2)
```

Rounds answers to 2 decimal places.

# ⚠ Important SQL Trick

### Using AVG for Percentages

Instead of:

```sql
SUM(condition) / COUNT(*)
```

you can write:

```sql
AVG(
 CASE WHEN condition
 THEN 1
 ELSE 0
 END
)
```

This is a common SQL interview pattern.
