# 📌 Problem Statement

### Cinema Table

| Column Name | Type |
|------------|------|
| id | int |
| movie | varchar |
| description | varchar |
| rating | float |

### 📝 Description

- `id` is the primary key.
- Each row contains:
  - Movie ID
  - Movie Name
  - Description
  - Rating

---

# 🎯 Task

Report all movies that satisfy:

✅ Have an odd-numbered ID

✅ Description is not `"boring"`

Return the result ordered by:

```sql
rating DESC
```

(highest rating first)

---

# 💻 My SQL Solution

```sql
SELECT *
FROM Cinema
WHERE id % 2 = 1
AND description <> 'boring'
ORDER BY rating DESC;
```

---

# 🧠 Explanation of My Approach

To solve this problem:

1. Find movies with odd IDs.
2. Exclude movies whose description is `"boring"`.
3. Sort the remaining movies by rating in descending order.

---

# Step 1️⃣ Select All Columns

```sql
SELECT *
```

Returns all movie information.

---

# Step 2️⃣ Filter Odd IDs

```sql
id % 2 = 1
```

Using modulus (`%`):

| ID | ID % 2 |
|----|---------|
| 1 | 1 |
| 2 | 0 |
| 3 | 1 |
| 4 | 0 |
| 5 | 1 |

Only odd IDs remain.

---

# Step 3️⃣ Exclude Boring Movies

```sql
description <> 'boring'
```

`<>` means **not equal to**.

So movies whose description is `"boring"` are removed.

---

# Step 4️⃣ Sort by Rating

```sql
ORDER BY rating DESC
```

Higher rated movies appear first.

---
