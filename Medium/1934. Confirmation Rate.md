# 📌 Problem Statement

### Signups Table

| Column Name | Type |
|------------|------|
| user_id | int |
| time_stamp | datetime |

- `user_id` is unique.
- Stores users who signed up.

---

### Confirmations Table

| Column Name | Type |
|------------|------|
| user_id | int |
| time_stamp | datetime |
| action | ENUM |

### Possible Values of `action`

- `'confirmed'`
- `'timeout'`

Each row represents a confirmation request made by a user.

---

# 🎯 Task

For each user, calculate:

\[
\text{Confirmation Rate}
=
\frac{\text{Number of Confirmed Messages}}
{\text{Total Confirmation Requests}}
\]

:contentReference[oaicite:0]{index=0}

### Special Case

If a user never requested a confirmation message:

```text
Confirmation Rate = 0
```

Round the answer to **2 decimal places**.

---

# 💻 My SQL Solution

```sql
SELECT
    s.user_id,
    ROUND(
        AVG(
            IF(c.action = 'confirmed', 1, 0)
        ),
        2
    ) AS confirmation_rate
FROM Signups AS s
LEFT JOIN Confirmations AS c
ON s.user_id = c.user_id
GROUP BY s.user_id;
```

---

# 🧠 Explanation of My Approach

Instead of manually calculating:

```sql
confirmed_count / total_count
```

we can use a neat trick:

```sql
AVG(IF(action='confirmed',1,0))
```

Because:

- confirmed → 1
- timeout → 0

The average automatically becomes:

```text
sum(1s) / total rows
```

which is exactly the confirmation rate.

---

# Step 1️⃣ Join Both Tables

```sql
LEFT JOIN Confirmations c
ON s.user_id = c.user_id
```

We use **LEFT JOIN** because:

- Every user must appear.
- Some users may have no confirmation requests.

---

# Step 2️⃣ Convert Actions into Numbers

```sql
IF(c.action='confirmed',1,0)
```

Example:

| action | value |
|----------|--------|
| confirmed | 1 |
| timeout | 0 |

---

# Step 3️⃣ Take Average

```sql
AVG(...)
```

Example:

| Requests |
|-----------|
| 1 |
| 1 |
| 0 |

Average:

\[
\frac{1+1+0}{3}
=
1.00
\]

# Step 4️⃣ Round Answer

```sql
ROUND(...,2)
```

Rounds the result to 2 decimal places.

# ⚠ Important SQL Trick

Instead of:

```sql
SUM(...)
/
COUNT(...)
```

you can write:

```sql
AVG(IF(condition,1,0))
```

This is a very common SQL interview pattern.

# ⏱ Complexity Analysis

| Operation | Complexity |
|------------|-------------|
| JOIN | O(n + m) |
| GROUP BY | O(n) |

Overall:

```text
O(n + m)
```

Where:

- `n` = rows in Signups
- `m` = rows in Confirmations
