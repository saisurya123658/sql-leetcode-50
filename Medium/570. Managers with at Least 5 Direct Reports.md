# 📌 Problem Statement

### Employee Table

| Column Name | Type |
|------------|------|
| id | int |
| name | varchar |
| department | varchar |
| managerId | int |

### 📝 Description

- `id` is the primary key.
- Each employee may have a manager.
- `managerId` references the manager's employee ID.
- If `managerId` is `NULL`, the employee has no manager.

---

# 🎯 Task

Find the names of managers who have:

✅ At least **5 direct reports**

Return the result table in any order.

---

# 💻 My SQL Solution

```sql
SELECT name
FROM Employee
WHERE id IN
(
    SELECT managerId
    FROM Employee
    GROUP BY managerId
    HAVING COUNT(managerId) > 4
);
```

---

# 🧠 Explanation of My Approach

The idea is:

1. Count how many employees report to each manager.
2. Keep managers having at least 5 employees.
3. Retrieve the manager names using their IDs.

---

# Step 1️⃣ Group Employees by Manager

```sql
GROUP BY managerId
```

This creates groups for every manager.

Example:

| managerId | Employees |
|------------|-----------|
| 101 | Dan, James, Amy, Anne, Ron |

---

# Step 2️⃣ Count Direct Reports

```sql
COUNT(managerId)
```

Counts how many employees report to each manager.

Example:

| managerId | Count |
|------------|-------|
| 101 | 5 |

---

# Step 3️⃣ Keep Managers with 5+ Reports

```sql
HAVING COUNT(managerId) > 4
```

Equivalent to:

```text
COUNT >= 5
```

Only managers with at least 5 direct reports remain.

---

# Step 4️⃣ Get Manager Names

```sql
WHERE id IN (...)
```

The subquery returns manager IDs.

The outer query fetches their names.

# ⚠ Important SQL Concept

## Why HAVING Instead of WHERE?

❌ Incorrect:

```sql
WHERE COUNT(managerId) > 4
```

`WHERE` cannot filter aggregated values.

✅ Correct:

```sql
HAVING COUNT(managerId) > 4
```

`HAVING` is used after grouping.

---

# Alternative Solution (Self Join)

```sql
SELECT e1.name
FROM Employee e1
JOIN Employee e2
ON e1.id = e2.managerId
GROUP BY e1.id, e1.name
HAVING COUNT(*) >= 5;
```

This solution is also accepted.
